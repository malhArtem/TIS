## Часть 1. Python: Чтение трассировки стека

### ТЕОРИЯ 1.1: Анатомия сообщения об ошибке в Python

Когда программа на Python падает, она оставляет **traceback** — посмертную записку. Давайте научимся её читать.

```
Traceback (most recent call last):
  File "example.py", line 4, in <module>
    func()
  File "example.py", line 2, in func
    print(x)
NameError: name 'x' is not defined
```

**Что здесь написано?**
1. **Первая строка:** `Traceback (most recent call last):` — "Я покажу вам, как программа дошла до жизни такой, начиная с конца".
2. **Строки с отступом:** Каждая строка — это один шаг программы (один вызов функции). Читать нужно **снизу вверх**:
   - Сначала программа была в функции `func` (строка 2 файла) — там и случилась беда.
   - Потом мы смотрим выше: функцию `func` вызвали из модуля (строка 4).
3. **Последняя строка:** Тип ошибки (`NameError`) и причина (`name 'x' is not defined`).

**Важно:** Самая нижняя строка из стека — это место, где произошла ошибка. Самая верхняя (последняя в выводе) — что именно случилось.

## Часть 2. C++ (общие принципы): Чтение сообщений компилятора

### ТЕОРИЯ 2.1: Анатомия сообщения об ошибке в C++ (g++/clang)

Сообщения компилятора C++ могут выглядеть страшно, но структура у них простая.

```
example.cpp: In function 'int main()':
example.cpp:4:5: error: expected ';' before 'std'
     std::cout << a;
     ^~~~
```

**Что здесь написано?**
- `example.cpp:4:5` — файл, строка 4, столбец 5 (примерное место ошибки).
- `error: expected ';' before 'std'` — что пошло не так: ожидалась точка с запятой перед `std`.
- `std::cout << a;` — строка с кодом.
- `^~~~` — указатель на проблемное место.

**Важно:** Иногда реальная ошибка находится выше по коду. Например, если вы забыли точку с запятой на строке 4, компилятор может указать на строку 5, потому что только там он понял, что что-то не так.

## Часть 3. C++ (MSVC): Особенности Visual Studio

### ТЕОРИЯ 3.1: Анатомия сообщений MSVC

Компилятор Microsoft Visual C++ (MSVC) имеет свою систему кодировки ошибок.

```
1>test.cpp(5,5): error C2143: syntax error: missing ';' before 'cout'
```

**Что здесь написано?**
- `1>` — номер проекта/потока при сборке.
- `test.cpp(5,5)` — файл, строка 5, столбец 5.
- `error C2143` — код ошибки (C — compiler, LNK — linker).
- `syntax error: missing ';' before 'cout'` — описание.

**Категории ошибок MSVC:**
- **Cxxxx** — ошибки компилятора (например, C2065, C2143, C2440)
- **LNKxxxx** — ошибки компоновщика (например, LNK2019, LNK2005)
- **E****** — ошибки IntelliSense (подсказки в редакторе, не влияют на сборку)

**Важно:** В Visual Studio всегда есть два источника "ошибок": красные подчёркивания IntelliSense и реальные ошибки сборки. Не верьте IntelliSense на 100% — ориентируйтесь на список ошибок после сборки.

   - `#pragma once` защищает от включения в одном файле, но не в разных. Что происходит, когда `config.h` включается и в `network.cpp`, и в `main.cpp`?
4. Исправьте проблему:
   - В `config.h` замените `int timeout = 30;` на `extern int timeout;` (объявление).
   - В `main.cpp` добавьте определение: `int timeout = 30;` (до `main`).
   - В `network.cpp` оставьте `#include "config.h"` — он увидит объявление.

### ТЕОРИЯ 4.1: Стратегия отладки

Когда в коде много ошибок, важно соблюдать порядок:
1. **Исправляйте ошибки сверху вниз.** Первая ошибка в списке часто вызывает остальные.
2. **После каждого исправления перекомпилируйте/перезапускайте.** Иногда одно исправление убирает 10 ошибок.
3. **Читайте полное сообщение.** Не только "error", но и "note:", "warning:" — они дают подсказки.
4. **Если не понимаете ошибку — загуглите.** Но сначала попробуйте проанализировать сами.
