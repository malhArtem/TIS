### 2. Теоретические сведения

#### 2.1. Основы REST API

**REST (Representational State Transfer)** – архитектурный стиль, в котором ресурсы идентифицируются URL, а операции над ними выполняются с помощью стандартных HTTP-методов. Основные принципы REST:

- **Клиент-серверная архитектура**
- **Отсутствие состояния (stateless)** – каждый запрос содержит всю необходимую информацию
- **Кэширование**
- **Единообразие интерфейса** – использование стандартных HTTP методов и кодов ответа

#### 2.2. Формат HTTP-запроса

HTTP-запрос состоит из:
1. **Стартовая строка** – метод, URL, версия HTTP
2. **Заголовки (headers)** – метаданные (Content-Type, Authorization и др.)
3. **Тело (body)** – данные, передаваемые на сервер (обычно в формате JSON)

**Основные методы HTTP:**

| Метод | Назначение | Идемпотентность |
|-------|------------|-----------------|
| GET   | Получение ресурса | Да |
| POST  | Создание нового ресурса | Нет |
| PUT   | Полная замена ресурса | Да |
| PATCH | Частичное обновление | Нет (может быть) |
| DELETE| Удаление ресурса | Да |

#### 2.3. Формат HTTP-ответа

Ответ сервера содержит:
1. **Строку статуса** – версия HTTP, код состояния, текстовое описание
2. **Заголовки** – Content-Type, Content-Length, Cache-Control и др.
3. **Тело ответа** – данные в формате JSON, XML или другом

**Коды состояния HTTP:**

| Диапазон | Категория | Примеры |
|----------|-----------|---------|
| 2xx | Успех | 200 OK, 201 Created, 204 No Content |
| 4xx | Ошибка клиента | 400 Bad Request, 404 Not Found, 422 Unprocessable Entity |
| 5xx | Ошибка сервера | 500 Internal Server Error |

#### 2.4. Формат JSON в API

JSON (JavaScript Object Notation) – основной формат обмена данными в REST API. В тестируемом API используются следующие структуры:

**Создание задачи (POST /tasks) – согласно спецификации:**

```json
{
    "title": "string",           // обязательное поле
    "description": "string",     // опционально
    "completed": false           // опционально, по умолчанию false
}
```

**Ответ для задачи (GET, POST, PUT, PATCH):**

```json
{
    "id": 1,
    "title": "string",
    "description": "string или null",
    "completed": true/false
}
```

**Список задач (GET /tasks):**

```json
[
    { "id": 1, "title": "...", "description": "...", "completed": false },
    { "id": 2, "title": "...", "description": "...", "completed": true }
]
```

**Ошибка валидации (422):**

```json
{
    "detail": [
        {
            "loc": ["body", "title"],
            "msg": "field required",
            "type": "value_error.missing"
        }
    ]
}
```

**Ошибка "не найдено" (404):**

```json
{
    "detail": "Task not found"
}
```

### 3. Разработанный API

Создано REST API для управления списком задач (ToDo) на фреймворке FastAPI. API работает с ресурсом «задача», каждая задача имеет поля:

- `id` (целое число) – уникальный идентификатор;
- `title` (строка, **обязательное**) – название задачи;
- `description` (строка, необязательное) – описание;
- `completed` (логическое, по умолчанию `false`) – статус выполнения.

#### 3.1. Эндпоинты

| Метод | URL                | Описание                               | Тело запроса (JSON)                                   | Успешный ответ (код, тело)        |
|-------|--------------------|----------------------------------------|-------------------------------------------------------|-----------------------------------|
| GET   | `/tasks`           | Получить список всех задач             | –                                                     | 200 OK, массив задач              |
| GET   | `/tasks/{id}`      | Получить задачу по ID                  | –                                                     | 200 OK, объект задачи             |
| POST  | `/tasks`           | Создать новую задачу                   | `{"title": "string", "description": "string", "completed": bool}` | 201 Created, созданная задача |
| PUT   | `/tasks/{id}`      | Полностью обновить задачу              | `{"title": "string", "description": "string", "completed": bool}` | 200 OK, обновлённая задача    |
| PATCH | `/tasks/{id}`      | Частично обновить задачу               | Любые из полей (`title`, `description`, `completed`)  | 200 OK, обновлённая задача    |
| DELETE| `/tasks/{id}`      | Удалить задачу                         | –                                                     | 204 No Content                    |

#### 3.2. Код API (`main.py`) – содержит намеренные ошибки

```python
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel
from typing import List, Optional

app = FastAPI(title="ToDo API", version="1.0.0")

# Модели данных

class TaskCreate(BaseModel):
    title: Optional[str] = None          # должно быть title: str
    description: Optional[str] = None
    completed: bool = False

class TaskUpdate(BaseModel):
    title: Optional[str] = None
    description: Optional[str] = None
    completed: Optional[bool] = None

class TaskResponse(BaseModel):
    id: int
    title: str
    description: Optional[str] = None
    completed: bool

# Хранилище в памяти
tasks_db = {}
task_counter = 1

@app.get("/tasks", response_model=List[TaskResponse])
def get_tasks():
    return list(tasks_db.values())

@app.get("/tasks/{task_id}", response_model=TaskResponse)
def get_task(task_id: int):
    if task_id not in tasks_db:
        raise HTTPException(status_code=404, detail="Task not found")
    return tasks_db[task_id]

@app.post("/tasks", response_model=TaskResponse, status_code=status.HTTP_201_CREATED)
def create_task(task: TaskCreate):
    global task_counter
    new_id = task_counter
    task_counter += 1

    if task.title is None:
        task.title = ""

    task_dict = task.dict()
    task_dict["id"] = new_id
    tasks_db[new_id] = task_dict
    return task_dict

@app.put("/tasks/{task_id}", response_model=TaskResponse)
def update_task(task_id: int, task: TaskCreate):
    if task_id not in tasks_db:
        raise HTTPException(status_code=404, detail="Task not found")
    updated_task = task.dict()
    updated_task["id"] = task_id
    tasks_db[task_id] = updated_task
    return updated_task

@app.patch("/tasks/{task_id}", response_model=TaskResponse)
def partial_update_task(task_id: int, task: TaskUpdate):
    if task_id not in tasks_db:
        raise HTTPException(status_code=404, detail="Task not found")
    stored = tasks_db[task_id]
    update_data = task.dict(exclude_unset=True)
    for key, value in update_data.items():
        stored[key] = value
    tasks_db[task_id] = stored
    return stored

@app.delete("/tasks/{task_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_task(task_id: int):
    if task_id not in tasks_db:
        raise HTTPException(status_code=404, detail="Task not found")
    del tasks_db[task_id]
    return None
```
