## 4. Подготовка: запуск тестового API

### 4.1. Создание файла сервера

Создайте на своём компьютере файл с именем `main.py` и скопируйте в него следующий код:

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import Optional, List
from datetime import datetime

app = FastAPI(title="Student API (тестовый)")

# ========== Модели данных ==========
class Student(BaseModel):
    id: int
    name: str
    email: str
    age: int
    is_active: bool = True
    enrolled_at: datetime = datetime.now()

class StudentCreate(BaseModel):
    name: str
    email: str
    age: int

class Course(BaseModel):
    id: int
    title: str
    credits: int

# ========== Хранилища данных ==========
students_db = {
    1: Student(id=1, name="Иван Петров", email="ivan@example.com", age=20),
    2: Student(id=2, name="Мария Сидорова", email="maria@example.com", age=22),
}
courses_db = {
    101: Course(id=101, title="Тестирование ПО", credits=4),
    102: Course(id=102, title="Базы данных", credits=3),
}

# ========== Эндпоинты ==========

@app.get("/students", response_model=List[Student])
async def get_all_students():
    """Получить всех студентов"""
    return list(students_db.values())

@app.get("/students/{student_id}")
async def get_student(student_id: int):
    """Получить студента по ID"""
    if student_id not in students_db:
        return {}
    return students_db[student_id]

@app.post("/students", status_code=201)
async def create_student(student: StudentCreate):
    """Создать нового студента"""
    new_id = max(students_db.keys()) + 1 if students_db else 1
    new_student = Student(
        id=new_id,
        name=student.name,
        email=student.email,
        age=student.age
    )
    students_db[new_id] = new_student
    return new_student

@app.delete("/students/{student_id}")
async def delete_student(student_id: int):
    """Удалить студента"""
    if student_id not in students_db:
        return {"error": "Student not found"}
    del students_db[student_id]
    return {"message": "Student deleted"}

@app.get("/students/{student_id}/courses")
async def get_student_courses(student_id: int):
    """Получить курсы студента"""
    if student_id not in students_db:
        return []
    if student_id % 2 == 1:
        return list(courses_db.values())
    return []

@app.patch("/students/{student_id}")
async def update_student(student_id: int, name: Optional[str] = None, age: Optional[int] = None):
    """Обновить данные студента"""
    if student_id not in students_db:
        raise HTTPException(status_code=404, detail="Student not found")
    if name:
        students_db[student_id].name = name
    if age:
        students_db[student_id].age = age
    return students_db[student_id]
```

## 6. Теоретические сведения (кратко)

### Типичные дефекты документации API

| Тип дефекта | Пример |
|-------------|--------|
| Неверный статус-код | Ожидается `404`, реально `200` |
| Отсутствие поля в ответе | Документация описывает `name`, реально `full_name` |
| Лишнее поле в ответе | Реальное API возвращает `created_at`, в документации нет |
| Неверная обязательность поля | Поле помечено как обязательное, но сервер не требует его |
| Отсутствующая ошибка | Документация описывает `409 Conflict`, но сервер никогда не возвращает этот код |

### Серьёзность дефектов

| Серьёзность | Описание |
|-------------|----------|
| **High** | Документация содержит неверную информацию о критическом сценарии |
| **Medium** | Документация не описывает важные коды ошибок |
| **Low** | Незначительные неточности, не влияющие на использование |

## Приложение Б. Как выполнять HTTP-запросы через браузер (инструкция для студента)

Для выполнения лабораторной работы вам потребуется отправлять запросы разных типов: `GET`, `POST`, `PATCH`, `DELETE`. Ниже описаны **три способа** сделать это без установки дополнительного ПО.

---

### Способ 1. Адресная строка (только для GET-запросов)

Это самый простой способ, но он работает **только для GET-запросов**.

**Что делать:**
1. Откройте браузер.
2. Введите URL в адресную строку.
3. Нажмите Enter.

**Пример:**
```
http://localhost:8000/students
```
```
http://localhost:8000/students/1
```

**Результат:** JSON-ответ отобразится прямо в окне браузера.

**Ограничение:** так нельзя отправить POST, PATCH или DELETE.

---

### Способ 2. Консоль разработчика (для любых запросов)

Этот способ работает во всех современных браузерах (Chrome, Firefox, Edge) и позволяет отправлять любые типы запросов.

#### Шаг 1. Открыть консоль разработчика

| Браузер | Как открыть |
|---------|--------------|
| Chrome / Edge | Нажмите `F12` или `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac), затем перейдите на вкладку **Console** |
| Firefox | Нажмите `F12` или `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac), затем перейдите на вкладку **Консоль** |

#### Шаг 2. Ввести команду fetch()

Используйте JavaScript-функцию `fetch()` для отправки запросов.

---

**Пример 1: GET-запрос**

```javascript
fetch('http://localhost:8000/students')
  .then(response => response.json())
  .then(data => console.log(data))
```

Скопируйте эту команду в консоль и нажмите Enter. Ответ появится в консоли.

---

**Пример 2: POST-запрос (создание студента)**

```javascript
fetch('http://localhost:8000/students', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Тест Тестов',
    email: 'test@example.com',
    age: 20
  })
})
.then(response => {
  console.log('Статус:', response.status);
  return response.json();
})
.then(data => console.log('Ответ:', data));
```

---

**Пример 3: PATCH-запрос (обновление)**

```javascript
fetch('http://localhost:8000/students/1', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Обновлённое имя'
  })
})
.then(response => {
  console.log('Статус:', response.status);
  return response.json();
})
.then(data => console.log('Ответ:', data));
```

---

**Пример 4: DELETE-запрос (удаление)**

```javascript
fetch('http://localhost:8000/students/2', {
  method: 'DELETE'
})
.then(response => {
  console.log('Статус:', response.status);
  return response.json();
})
.then(data => console.log('Ответ:', data));
```

---

#### Как узнать статус-код ответа?

В примерах выше статус выводится в консоль. Обратите внимание на строку:
```javascript
console.log('Статус:', response.status);
```

После выполнения запроса в консоли появится что-то вроде:
```
Статус: 200
Ответ: {id: 3, name: "Тест Тестов", ...}
```

---

### Способ 3. Онлайн-сервис reqbin.com (для любых запросов)

Если консоль кажется сложной, используйте бесплатный онлайн-инструмент.

**Инструкция:**

1. Откройте сайт: **https://reqbin.com**
2. Настройте запрос:
   - В поле **URL** введите `http://localhost:8000/students`
   - Выберите метод (GET, POST, PATCH, DELETE) из выпадающего списка
   - Для POST/PATCH выберите **Content-Type: application/json** и введите JSON в поле **Content**
3. Нажмите **Send**
4. Посмотрите статус-код и тело ответа

**Важно:** reqbin.com работает с локальным сервером (`localhost`), потому что запрос отправляется из вашего браузера на ваш же компьютер.

```javascript
fetch('http://localhost:8000/students/999', {method:'DELETE'}).then(r=>{console.log('Статус:',r.status);return r.json();}).then(console.log)
```

### PATCH /students/1 (обновить имя)
```javascript
fetch('http://localhost:8000/students/1', {method:'PATCH',headers:{'Content-Type':'application/json'},body:JSON.stringify({name:'Новое имя'})}).then(r=>{console.log('Статус:',r.status);return r.json();}).then(console.log)
```

## Частые ошибки и их решение

| Проблема | Решение |
|----------|---------|
| Сервер не отвечает | Проверьте, запущен ли сервер: `http://localhost:8000/students` должен показывать JSON |
| Ошибка CORS | Не возникает при работе с localhost |
| Синтаксическая ошибка в консоли | Убедитесь, что скопировали всю команду целиком |
| POST запрос не работает | Проверьте, что в теле запроса JSON без лишних запятых |
| Не вижу статус ответа | Используйте полную версию команды с `console.log('Статус:', response.status)` |

## Что записывать в отчёт?

При выполнении каждого запроса фиксируйте:

1. **Какой запрос отправили** (метод и URL, тело для POST/PATCH)
2. **Какой статус-код получили** (например, 200, 404, 422)
3. **Какое тело ответа получили** (скопируйте JSON)

Пример записи:
```
Запрос: GET http://localhost:8000/students/999
Статус: 200
Тело ответа: {}
Ожидание по документации: статус 404
Несоответствие: ДА (документация обещает 404, реально 200)
```
