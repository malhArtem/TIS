## Теоретическая часть (расширенная): Как читать сообщения об ошибках

### 1. Ошибки в Python

Когда программа на Python завершается аварийно, интерпретатор выводит **traceback** (трассировку стека). Он показывает путь выполнения программы до момента возникновения исключения.

#### Структура traceback

```
Traceback (most recent call last):
  File "путь_к_файлу", line номер_строки, in имя_функции
      код_вызвавший_ошибку
ТипИсключения: пояснение
```

**Пример 1. NameError**

```python
# example1.py
def func():
    print(x)

func()
```

Запуск:
```
Traceback (most recent call last):
  File "example1.py", line 4, in <module>
    func()
  File "example1.py", line 2, in func
    print(x)
NameError: name 'x' is not defined
```

**Как читать:**
- Первая строка: `Traceback (most recent call last):` — начало трассировки.
- Затем идут строки с информацией о каждом вызове функции (стек вызовов). Самая последняя запись (внизу) — место, где возникла ошибка.
- В данном примере:
  - Сначала был вызов `func()` в модуле (строка 4).
  - Внутри `func` на строке 2 произошла ошибка.
  - Тип исключения: `NameError`.
  - Пояснение: `name 'x' is not defined` — переменная `x` не определена.

**Пример 2. SyntaxError**

```python
# example2.py
if True
    print("True")
```

Запуск:
```
  File "example2.py", line 1
    if True
           ^
SyntaxError: invalid syntax
```

**Как читать:**
- Указан файл и строка (1).
- Символ `^` указывает точное место, где интерпретатор обнаружил проблему (после `if True` ожидается двоеточие).
- Тип ошибки: `SyntaxError`, описание: `invalid syntax`.

**Пример 3. TypeError**

```python
# example3.py
a = 5
b = "10"
print(a + b)
```

Запуск:
```
Traceback (most recent call last):
  File "example3.py", line 3, in <module>
    print(a + b)
TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

**Как читать:**
- Ошибка на строке 3.
- Тип: `TypeError`.
- Пояснение: операция `+` не поддерживается между целым числом (`int`) и строкой (`str`).

**Пример 4. IndexError**

```python
# example4.py
lst = [1, 2, 3]
print(lst[3])
```

Запуск:
```
Traceback (most recent call last):
  File "example4.py", line 2, in <module>
    print(lst[3])
IndexError: list index out of range
```

**Как читать:**
- Ошибка на строке 2.
- Тип: `IndexError`.
- Пояснение: индекс списка вне допустимого диапазона (список содержит элементы с индексами 0,1,2).

**Пример 5. KeyError**

```python
# example5.py
d = {"name": "Alice"}
print(d["age"])
```

Запуск:
```
Traceback (most recent call last):
  File "example5.py", line 2, in <module>
    print(d["age"])
KeyError: 'age'
```

**Как читать:**
- Ошибка на строке 2.
- Тип: `KeyError`.
- Пояснение: ключ `'age'` отсутствует в словаре.

**Пример 6. AttributeError**

```python
# example6.py
x = None
print(x.upper())
```

Запуск:
```
Traceback (most recent call last):
  File "example6.py", line 2, in <module>
    print(x.upper())
AttributeError: 'NoneType' object has no attribute 'upper'
```

**Как читать:**
- Ошибка на строке 2.
- Тип: `AttributeError`.
- Пояснение: у объекта типа `NoneType` нет метода `upper()`.

#### Общие правила для Python
- Внимательно читайте **последнюю строку** traceback — она содержит тип исключения и причину.
- Строка, указанная непосредственно перед типом исключения, — место возникновения ошибки.
- Если ошибка синтаксическая, ищите символ `^` — он указывает на проблемное место.
- Иногда traceback содержит несколько вложенных вызовов — двигайтесь сверху вниз, чтобы понять последовательность.

### 2. Ошибки в C++

Сообщения компилятора C++ (например, g++, clang++) могут выглядеть пугающе, особенно при работе с шаблонами. Однако структура у них типична.

#### Ошибки компиляции (compile-time errors)

**Пример 7. Пропущена точка с запятой**

```cpp
// example7.cpp
#include <iostream>
int main() {
    int a = 5
    std::cout << a;
    return 0;
}
```

Компиляция g++:
```
example7.cpp: In function 'int main()':
example7.cpp:4:5: error: expected ';' before 'std'
     std::cout << a;
     ^~~
```

**Как читать:**
- `example7.cpp:4:5` — файл, строка 4, столбец 5.
- `error: expected ';' before 'std'` — описание: ожидалась точка с запятой перед `std`.
- Далее показана строка с ошибкой и указатель `^~~` примерно на место.

**Пример 8. Необъявленный идентификатор**

```cpp
// example8.cpp
#include <iostream>
int main() {
    int x = 10;
    y = x + 5;
    std::cout << y;
    return 0;
}
```

Компиляция:
```
example8.cpp: In function 'int main()':
example8.cpp:4:5: error: 'y' was not declared in this scope
     y = x + 5;
     ^
```

**Как читать:**
- Строка 4, столбец 5.
- `'y' was not declared in this scope` — переменная `y` не объявлена в этой области видимости.

**Пример 9. Несоответствие типов**

```cpp
// example9.cpp
#include <iostream>
#include <string>
int main() {
    std::string s = "123";
    int num = s;
    std::cout << num;
    return 0;
}
```

Компиляция (g++):
```
example9.cpp: In function 'int main()':
example9.cpp:5:14: error: conversion from 'std::__cxx11::string' {aka 'std::__cxx11::basic_string<char>'} to non-scalar type 'int' requested
     int num = s;
              ^
```

**Как читать:**
- Строка 5, столбец 14.
- Ошибка преобразования: из типа `std::string` в тип `int` (нескалярный тип).
- Компилятор показывает, что пытается преобразовать `s` в `int`, но это невозможно без явного преобразования.

**Пример 10. Ошибка, связанная с шаблонами** (длинное сообщение)

```cpp
// example10.cpp
#include <vector>
int main() {
    std::vector<int> v;
    v.push_back("hello"); // передаём строку вместо int
    return 0;
}
```

Сообщение может быть очень длинным, но ключевая информация обычно находится в начале или в конце:
```
example10.cpp:4:18: error: no matching function for call to 'std::vector<int>::push_back(const char [6])'
     v.push_back("hello");
                  ^~~~~~~
/usr/include/c++/9/bits/stl_vector.h:1184:7: note: candidate: 'void std::vector<_Tp, _Alloc>::push_back(const value_type&) [with _Tp = int; _Alloc = std::allocator<int>; value_type = int]'
 1184 |       push_back(const value_type& __x)
      |       ^~~~~~~~~
...
```

**Как читать:**
- Первая строка указывает на проблемное место в вашем коде (example10.cpp:4:18).
- Далее компилятор пытается объяснить, какие перегрузки функции рассматривал и почему они не подходят.
- **Совет:** ищите первое упоминание вашего файла и строки — это источник ошибки. Остальное — технические детали, которые можно анализировать при необходимости.

#### Ошибки компоновки (linker errors)

Возникают, когда все единицы трансляции скомпилированы успешно, но при сборке итогового исполняемого файла не найдены определения функций или переменных.

**Пример 11. Undefined reference**

```cpp
// file.h
void foo(); // объявление

// main.cpp
#include "file.h"
int main() {
    foo(); // вызов
    return 0;
}
```

Компиляция и компоновка (g++ main.cpp):
```
/usr/bin/ld: /tmp/ccJkR5aH.o: в функции main:
main.cpp:(.text+0x14): неопределённая ссылка на foo()
collect2: error: ld returned 1 exit status
```

**Как читать:**
- `неопределённая ссылка на foo()` — функция `foo` объявлена, но не определена (нет тела).
- Указан объектный файл и функция `main`, где произошёл вызов.
- Решение: либо добавить определение `foo` в один из .cpp файлов, либо скомпилировать все файлы вместе.

**Пример 12. Multiple definition**

```cpp
// config.h
int value = 10; // определение в заголовке

// a.cpp
#include "config.h"
void funcA() { value = 5; }

// b.cpp
#include "config.h"
void funcB() { value = 7; }
```

Компиляция (g++ a.cpp b.cpp):
```
/usr/bin/ld: /tmp/ccaBx2Hl.o:(.data+0x0): multiple definition of `value'; /tmp/ccR8c4mC.o:(.data+0x0): first defined here
collect2: error: ld returned 1 exit status
```

**Как читать:**
- `multiple definition of 'value'` — переменная `value` определена многократно.
- Указаны объектные файлы, в которых найдены определения.
- Решение: оставить определение только в одном .cpp файле, а в заголовке сделать `extern int value;`.

#### Общие правила для C++
- Внимательно читайте первую строку ошибки — она содержит файл, строку и краткое описание.
- Не игнорируйте предупреждения (warnings) — часто они указывают на потенциальные ошибки.
- Для длинных сообщений от шаблонов ищите первое вхождение вашего кода, остальное — внутренности библиотек.
- Если ошибка компоновки, проверьте, все ли .cpp файлы переданы компилятору, и нет ли множественных определений.
